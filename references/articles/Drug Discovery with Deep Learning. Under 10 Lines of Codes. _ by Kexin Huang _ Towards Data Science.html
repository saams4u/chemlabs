Drug Discovery with Deep Learning. Under 10 Lines of Codes.

Introduce DeepPurpose, a deep learning-based toolkit for drug-target interaction prediction, drug repurposing, and virtual screening.

Kexin Huang

Jul 8, 2020Â·9 min read

TL;DR: Deep learning is revolutionizing the drug discovery industry. In this post, I show how to use the DeepPurpose toolkit to unlock 50+ models for Drug-Target Interaction (DTI) prediction, a fundamental task in drug discovery, using only a few lines of code in a scikit-learn fashion. DeepPurpose also enables an easy interface to do two important applications of DTI prediction: virtual screening and drug repurposing.

Introduction

Discover a new drug takes more than 10 years and costs higher than $2.6 billion [1]. Recently, many AI for drug discovery startups emerge and have successfully applied deep learning techniques to aid drug discovery research and greatly shorten time/save cost [2,3]. Thus, it is a very exciting and booming field ðŸ’¥!

However, the interest within the data scientist community in deep learning for drug discovery, although growing quickly, is still far behind other deep learning applications such as NLP and images. One possible reason is that the lack of open-sourced tools prevents many data scientists to probe into this exciting domain. Existing toolkits such as DeepChem, OpenChem, MoleculeNet and etc., have democratized the use of deep learning for life sciences greatly. In this post, I describe a new addition called DeepPurpose that works specifically for drug-target interaction prediction, a fundamental task for the drug discovery process. All data scientists love scikit-learn. Thus, we design DeepPurpose as a scikit-learn-fashion framework that wraps 50+ models in less than 10 lines ðŸ˜‡!

Background

Letâ€™s start with some biochemistry concepts. A major category of disease is usually attributed to the target protein in the disease pathway. A drug is discovered to modulate the target protein. Through this modulation of disease responsible protein, the disease can be cured. One of the major paradigms of the drug action mechanism is the â€˜Lock-And-Keyâ€™ theory [4]. A protein is a â€œ lockâ€ ðŸ”’ and drug discovery is to find the right â€œkeyâ€ ðŸ”‘ to unlock the target (i.e., the right drug to modulate the protein). This fitness is called binding affinity.

Drug-target interaction (DTI) measures the binding affinity of drug molecules to the protein targets. Thus, we can easily imagine that an accurate DTI deep learning model can greatly benefit the drug discovery process [5]. More specifically, virtual screening and drug repurposing are two main applications based on DTI. Virtual screening helps identify ligand candidates that can bind to the protein of interest, whereas drug repurposing finds new therapeutic purposes for existing drugs.

DeepPurpose Overview

DeepPurpose formulates DTI models as an encoder-decoder framework, which includes a majority of previous works [6,7,8,9,10] and much more. Two encoders generate embeddings for drug and protein respectively and then they are concatenated into a decoder, where it predicts the binding affinity score.

DeepPurpose uses the most accessible input data format. The input is a drug target pair, where the drug uses the simplified molecular-input line-entry system (SMILES) string and the target uses the amino acid sequence. The output is a score indicating the binding activity of the drug-target pair.

For drugs, we provide 8 encoders: four classic cheminformatics fingerprints (Morgan, Pubchem, Daylight, and RDKit 2D) with Deep Neural Network (DNN) on top of them; 1D Convolutional Neural Network (CNN) on SMILES string characters; Recurrent Neural Network on top of CNN to leverage the sequential order; Transformer encoder on sub-structure partition fingerprint [11]; and Message passing neural network on the molecular graphs.

For the target protein, we have 7 encoders: DNN on top of four classic computational biology fingerprints (AAC, PseAAC, Conjoint Triad, Quasi-sequence); CNN on the amino acid sequence; RNN on top of CNN; and transformer on the sub-sequence fingerprint.

In combination, there are 56 (8 times 7) drug-target encoding models! Also, most of the models are novel ðŸ†•!

DeepPurpose Framework

Now after we have a conceptual overview of DTI and DeepPurpose, I begin to introduce the DeepPurpose programming framework. This framework consists of several steps where each step consists of one line of code:

  • Data loading
  • Encoder specification
  • Data encoding and split
  • Model configuration generation
  • Model initialization
  • Model training
  • Repurposing/Screening
  • Model saving and loading

Letâ€™s go over each step! For a better learning experience, it is recommended to follow these steps in your own Jupyter notebook file or going over the blog post notebook. To install DeepPurpose, you can use the DeepPurpose Binder cloud notebook (simply clicking the link!) or use the local DeepPurpose environment. The instructions for the installation can be found here, where you can also find video installation tutorials.

Data loading

DeepPurpose takes in a NumPy array of drugâ€™s SMILES strings (X_drugs), a NumPy array of target proteinâ€™s amino acid sequence (X_targets), and a NumPy array of labels (y), which can either be binary 0/1 indicating interaction outcome or a real number indicating binding affinity value. The input drug and target arrays should be paired, i.e. y[0] is the score for X_drugs[0] and X_targets[0]. DeepPurpose will automatically switch between the binary classification task or regression task depending on the dataset.

Besides transforming into NumPy arrays through some data wrangling on your own, DeepPurpose also provides benchmark dataset loaders (DAVIS/KIBA/BindingDB) to ease preprocessing. For example, in this post, we will use the DAVIS dataset:

Data encoding and split

Now, we need to prepare the data encoding scheme for different encoders, using the utils.data_process function. In this function, we can specify train/validation/test split fractions, random seed to ensure the same data split for reproducibility, and also supports data splitting methods such as cold_drug and cold_protein, which splits on drug/proteins for model robustness evaluation to test unseen drug/proteins. The function outputs train, validation, and test set Pandas data frames.
